# AutoPlay Bot

## Project Overview

The AutoPlay Bot is an accessibility-focused assistant designed to help users navigate and interact with rhythm games through voice commands and computer vision. The system combines speech recognition, a locally hosted large language model, text-to-speech synthesis, and object detection to both navigate game menus and assist with gameplay.

---

# Components

## Speech-to-Text Model (STT)

The speech-to-text model used in this project is **Whisper Small**. This model detects voice input from the user and converts it into text that can be processed by the system.

### Why Whisper Small?

* Can run locally on consumer hardware.
* Lower resource requirements compared to larger models.
* Provides sufficient accuracy for menu navigation commands.
* Reduces latency by avoiding cloud-based processing.

---

## Large Language Model (LLM)

The large language model used is **gpt-oss-20b**. The LLM interprets text generated by the STT model and determines which actions or tools should be executed.

### Responsibilities

* Process user commands.
* Determine intended actions.
* Call tools that navigate game menus.
* Generate responses for the text-to-speech system.

### Why gpt-oss-20b?

* Can be run locally.
* Small enough to operate on available hardware.
* Provides strong reasoning and tool-calling capabilities.

---

## Text-to-Speech (TTS)

The text-to-speech solution used is **MIKU TTS**.

After the LLM determines an action, MIKU TTS provides spoken feedback to the user.

### Example

If the LLM issues a command to move up in a game menu, MIKU TTS may respond with:

> "Moving up."

### Why MIKU TTS?

* Already hosted and available for use.
* Easy to integrate.
* The team are fans of Hatsune Miku and wanted to incorporate her voice into the project.

---

## Computer Vision Model

The computer vision component uses **YOLO (You Only Look Once)**.

YOLO will be trained using gameplay footage from the target rhythm game. Once trained, it will detect gameplay elements and determine which inputs should be executed based on the current game state.

### Responsibilities

* Detect notes and gameplay objects.
* Identify relevant screen locations.
* Trigger appropriate key presses.
* Assist with automated gameplay.

### Why YOLO?

* Team members already have experience using YOLO.
* Real-time object detection performance.
* Suitable for fast-paced rhythm game environments.
* Can run locally.

# System Architecture

## Voice Navigation Pipeline

```text
User Voice
    ↓
Speech-to-Text (Whisper Small)
    ↓
Text
    ↓
LLM (gpt-oss-20b)
    ↓
Directions / Tool Calls
    ↓
Text-to-Speech (MIKU TTS)
    ↓
Voice Feedback
    ↓
Game Executes Action
```

## Gameplay Automation Pipeline

```text
Gameplay Feed
    ↓
YOLO Object Detection
    ↓
Detect Notes & Locations
    ↓
Determine Inputs
    ↓
Press Corresponding Keys
```
